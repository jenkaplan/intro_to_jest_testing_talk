What is Snapshot Testing?

  • Allows you to quickly write testing for static data
  • If the data changes, you don't need to rewrite your test, Jest will update the test by running `-u` after your test command
  • This makes Jest super easy to use to test

How Snapshot testing works

  • You write a simple test with snapshot, then it saves that into a folder of snapshots.
  • Then when you test it, it compares the page to the snapshot.
  • If they match, it passes. If it doesn't match, it shows you the changes and you can run the test again with `-u` to update the snapshot.

What does a test look like?


          it('loads in pokemon list', () => {
            const tree = renderer.create().toJSON();
            expect(tree).toMatchSnapshot();
          });

First Error!

``` FAIL tests/app.test.js ● loads in pokemon list TypeError: Cannot read property 'names' of undefined ```

What that means

That was caused by my data being structured like this:


            const data = {
              names: [
                {
                    "url": "http://pokeapi.co/api/v2/pokemon/1/",
                    "name": "bulbasaur"
                },
                // shortened for length
              ]
            }

The solution is to passing in data

In your test, you want to pass in your data, so literally copy and paste the API response like this:


            const data = {
              names: [
                {
                    "url": "http://pokeapi.co/api/v2/pokemon/1/",
                    "name": "bulbasaur"
                },
                // shortened for length
              ]
            }

data having an array called names, that holds the json threw things off

And in your test add this:


            it('loads in pokemon list', () => {
              const tree = renderer.create().toJSON();
              expect(tree).toMatchSnapshot();
            });

Now lets try this again...

New Error: Not a Function

``` FAIL tests/app.test.js ● loads in pokemon list TypeError: getPokemon is not a function at App.componentDidMount (src/App.js:13:4) ```

So then we got to App.js line 13 and we see this:


            componentDidMount() {
              this.getPokemon();
            }

That is clearly a function...

Rage!

Now we need to turn this function into a function

In the test file, you want to add this outside of your test


            const getPokemon = () => {}

And we change the test to this:


            it('loads in pokemon list', () => {
              const tree = renderer.create().toJSON();
              expect(tree).toMatchSnapshot();
            });

Now we have a passing test!

Summary of the problem

  • Jest had a hard time accessing an array inside the json
  • Jest didn't understand that function called inside of componentDidMount() was a function, so we had to make this explicate

How to make Mocha work with Jest

Add this to your package.json


            "scripts": {
              "test-server": "mocha",
              "test-react": "jest",
              "test": "npm run test-server && npm run test-react"
            },

How to get Jest to work nicely with images, CSS, and node modules

Add this to your package.json


            "jest": {
                "moduleNameMapper": {
                  "\\.(jpg|jpeg|png|gif|eot|otf|webp|svg|ttf|woff|woff2|mp4|webm|wav|mp3|m4a|aac|oga)$": "/reactTests/__mocks__/fileMock.js",
                  "\\.(css|less)$": "/client/src/stylesheets/"
                },
                "testPathIgnorePatterns": [
                  "/node_modules/"
                ]
              },
