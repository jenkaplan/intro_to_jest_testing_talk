Why Jest?

  • Jest is built by Facebook
  • Top Google results for using Mocha with React is from 2015 or early 2016
  • Was neglected for a long time, but for the past year they completely rewrote it and now it is great
  • Front-end only testing is easy, testing API calls from the backend is more complex
  • Downside is there is not many docs with complex examples yet
  • I found it easy to write tests for front-end only code, but when I had to write my first tests with the API calls from the backend, it was a challenge

What is Snapshot Testing?

  • Allows you to quickly write testing for static data
  • If the data changes, you don't need to rewrite your test, Jest will update the test by running `-u` after your test command
  • This makes Jest super easy to use to test

How Snapshot testing works

  • You write a simple test with snapshot, then it saves that into a folder of snapshots.
  • Then when you test it, it compares the page to the snapshot.
  • If they match, it passes. If it doesn't match, it shows you the changes and you can run the test again with `-u` to update the snapshot.
