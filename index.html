Why Jest?

  • Most Mocha testing for React is from 2015 or early 2016
  • Jest is built by Facebook
  • Was neglected for a long time, but for the past year they completely rewrote it and now it is great
  • Downside is there is not many docs with complex examples yet

What is Snapshot Testing?

  • Allows you to quickly write testing for static data
  • If the data changes, you don't need to rewrite your test, it will do it by running `-u` after your test command

What does a test look like?


          it('loads in pokemon list', () => {
            const tree = renderer.create().toJSON();
            expect(tree).toMatchSnapshot();
          });

First Error!

``` FAIL reactTests/planTests/projectPlan.test.js ‚óè test to see if the project plan renders correctly TypeError: Cannot read property 'names' of undefined ```

What that means

That was caused by my data being structured like this:


            const data = {
              names: [
                {
                    "url": "http://pokeapi.co/api/v2/pokemon/1/",
                    "name": "bulbasaur"
                },
                // shortened for length
              ]
            }

The solution is to passing in data

In your test, you want to pass in your data, so literally copy and paste the API response like this:


            const data = {
              names: [
                {
                    "url": "http://pokeapi.co/api/v2/pokemon/1/",
                    "name": "bulbasaur"
                },
                // shortened for length
              ]
            }

data having an array called names, that holds the json threw things off

And in your test add this:


            it('loads in pokemon list', () => {
              const tree = renderer.create().toJSON();
              expect(tree).toMatchSnapshot();
            });
